# Ostrich Service Backend\n\nFlask backend for the Ostrich service technician mobile application.\n\n## Setup\n\n```bash\npip install -r requirements_flask.txt\npython main_service_swagger.py\n```\n\n## API Documentation\n\n- **Port**: 8002\n- **Interactive Docs**: http://localhost:8002/docs/\n- **Health Check**: http://localhost:8002/health\n\n## API Endpoints\n\n### Authentication\n- `POST /api/v1/auth/login` - Technician login with username/password\n- `POST /api/v1/auth/signup` - Register new technician\n- `POST /api/v1/auth/send-otp` - Send OTP to technician\n- `POST /api/v1/auth/verify-otp` - Verify OTP and authenticate\n\n### Dashboard\n- `GET /api/v1/dashboard/` - Technician dashboard with assigned tickets and stats\n\n### Tickets\n- `GET /api/v1/tickets/assigned` - Get tickets assigned to technician (with filters)\n- `GET /api/v1/tickets/{id}` - Get detailed ticket information\n- `PUT /api/v1/tickets/{id}/status` - Update ticket status and notes\n- `POST /api/v1/tickets/{id}/location` - Capture location for ticket\n\n### Notifications\n- `GET /api/v1/notifications/` - Get service-related notifications\n\n## Features\n\n- JWT Authentication with multiple methods\n- Ticket assignment and management\n- Field service operations\n- Location tracking\n- Work completion reporting\n- Real-time notifications\n- Database integration with fallback data\n- Interactive Swagger documentation\n\n## Demo Credentials\n\n- **Username**: demo.tech\n- **Password**: password123\n- **OTP**: 123456\n\n## Database\n\n- Connects to MySQL database with fallback to demo data\n- Multiple technician test scenarios\n- Rich ticket data for testing
